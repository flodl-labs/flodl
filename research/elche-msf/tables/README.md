# Eval-focused paper tables

Curated tables for the manuscript, each derived from upstream sweep
aggregators. Numbers are quoted from the canonical `aggregate.txt` /
`per_cell.csv` (or per-cell `report.md` for the Gate A / Gate D
sweeps that don't have aggregators yet) of each sweep — re-running
those aggregators / extract scripts regenerates the underlying
numbers; this directory is the human-readable rollup.

## Figures

Each table embeds the most informative figure(s) from its source
sweep's `data/<sweep>/analysis/` dir. The Gate A and Gate D tables
embed figures generated by single-purpose scripts in
`data/cpu-async-multiseed/fig.py` and `data/resnet56-cpu-async/fig.py`
respectively (the new sweeps don't have full aggregator + analyze.py
yet — see "known gaps" below). The cross-sweep Pearson scatter for
the framing-validity table is generated by
`tables/fig_framing_gates.py`.

To regenerate everything image-side from the current extracts:

```
python3 research/elche-msf/data/passive-observation/analyze.py
python3 research/elche-msf/data/relaxed-anchor/analyze.py
python3 research/elche-msf/data/cliff-bracket/analyze.py
python3 research/elche-msf/data/pareto-frontier/analyze.py
python3 research/elche-msf/data/cpu-async-multiseed/fig.py
python3 research/elche-msf/data/resnet56-cpu-async/fig.py
python3 research/elche-msf/tables/fig_framing_gates.py
```

## Known gaps for follow-up batches

- **ResNet-56 α=1.0 cohort is single-seed.** Gate D's α-comparison is
  not paired (4 vs 1). Recommended next batch: 3 more α=1.0 cells
  (seeds 2, 3, 4) at ResNet-56 cpu-async msf, ~6 h overnight at
  ~93 min/run.
- **Aggregators / analysis layer for Gate A and Gate D.** The two
  new sweep dirs ship with `extract.py` + `README.md` only. If the
  paper needs per-rank Pearson tables or by-k slope figures for
  these cohorts, add `aggregate.py` + `analyze.py` modeled on the
  existing sweeps.
